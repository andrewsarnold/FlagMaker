What is FlagMaker?

FlagMaker is a little tool I built to easily create flags without having to use complicated image editors like Inkscape or Photoshop, and includes most common flag elements.

In this version, you can create a flag with six different base types and then add as many overlays as you like. You can save your work to a proprietary file format (see below) or export to a PNG or SVG file.

FAQ

  • What colors are available? – Two pre-set palettes exist, taken from the FOTW website. One ("available colors") is FOTW's own standard palette, taken from this page. The second ("standard colors") is the British Navy's "Flags of All Nations" palette, taken from this page and converted to the closest matching RGB color. You can also click "Advanced" to pick any color you like.
  • How do I make just a solid color flag? – Just use a grid with size 1×1. Same thing for stripes; just use 1×n. Grids are pretty versatile.
  • How do I make a perfect circle / perfect diamond? – Use a height of 0.
  • The alignment on the star / pall is a bit off. – Yes, they need some tweaking to get them more accurate.
  • You should add ______ to FlagMaker. – If it's not listed in the planned features section, send me a message and I'll think about it.

The .flag file type

.flag is a file type you can use to save and open your flags. It's a plain text file so you can also use your favorite text editor to modify them (although FlagMaker doesn't include certain error handling, so if you make a mistake it's possible to crash the program). Feel free to look at the presets for some examples, but here's a rundown of the .flag file syntax:

name – Only used to fill the Presets menu. Put anything you want here.
ratio – The ratio of the flag's height to width. Separate with a colon.
gridsize – A multiple of the ratio, which determines how finely you can place your elements. Play around with the dropdown in the program to get a good handle on what this does.

division – Start defining the division. Only include once!
type – One of: grid, fesses, pales, bends forward, bends backward, bends both
color1 / color2 / color3 – Pretty self-explanatory. Use either RGB or ARGB formatted in hexadecimal, e.g. 822433 or fedd00 or cc284e36. No # sign. Sorry, no Pantone. (Converting Pantone to RGB is some kind of black magic that I don't understand at all.)
size1 / size2 / size3 – Slider values.

overlay – Start defining a new overlay. You can have as many of these sections as you like.
type – One of: canton, triangle, diamond, fimbriation forward, fimbriation backward, saltire, pall, cross, ellipse, star, crescent, hammer and sickle, equitorial cross, star seven, star of david, line horizontal, line vertical
color – Same as above.
size1 / size2 / size3 / size4 – Slider values.

The files should be backward-compatible (it'll generate a canton for any overlays not supported in your version). For some overlays, some values have been swapped around for consistency.
